# トップページ設計整理メモ（収益化メディアの入口）

トップページを「検索流入 → 商品ページ → 送客」の起点として整理するための設計メモ。今回は実装せず、構造と優先順位のみ記載。

---

## 1. 現在の構成

### サイトルート `/`（app/page.tsx）

- **内容**: Next.js デフォルトのプレースホルダー（「To get started, edit...」＋Vercel/Next リンク）。
- **データ**: なし。メディア入口としては未使用。
- **役割**: 実質なし。多くのユーザーは直リンクや検索で `/oliveyoung` や商品ページに流入する想定。

### Olive Young 入口 `/oliveyoung`（app/oliveyoung/page.tsx）

- **役割**: コメント上「メディア入口ページ」。ヒーロー・サマリー・注目商品・ブランド導線をまとめた**実質のトップ**。
- **利用データ**:
  - `getRankingRunDates()` → 直近 runDate
  - `getRankingTopNWithProducts(runDate, 3)` → 注目商品 TOP3（画像・名前・ブランド・商品詳細リンク）
  - `getBrandRankingByDate(runDate)` → 総ブランド数・NEW数・急上昇TOP5・NEWランクインブランド
  - `CATEGORY_SLUGS` / `CATEGORY_DISPLAY_NAMES` → カテゴリリンク（マスク・美容液・クリーム）
- **表示内容**:
  - ヒーロー: タイトル・説明・対象日・カテゴリリンク・「商品ランキングを見る」「ブランドランキングを見る」CTA
  - サマリーカード: 対象日・総商品数・総ブランド数・NEWブランド数
  - 今日の注目商品 TOP3（ランク・画像・名前・ブランド・商品詳細リンク）
  - 急上昇ブランド TOP5 / NEWランクインブランド（ブランド詳細リンク）
  - 導線カード: 商品ランキング一覧 / ブランドランキング一覧
- **遷移先**: 商品詳細 `/oliveyoung/products/[goodsNo]`、ブランド詳細 `/oliveyoung/brands/[runDate]/[brandKey]`、ランキング一覧 `/oliveyoung/rankings/[runDate]`、カテゴリ `/oliveyoung/category/[slug]`、ブランド一覧 `/oliveyoung/brands`。

## 2. トップページの役割定義

### A. SEO 入口としての役割

| 項目 | 方針 |
|------|------|
| **狙うキーワード** | 「韓国コスメ 人気」「Olive Young ランキング」「韓国コスメ オリーブヤング」など、ランキング・トレンド検索。現行の title/description と整合。 |
| **入口の候補** | サイトルート `/` に来た場合の受け皿を用意するか、`/` を `/oliveyoung` にリダイレクトして「トップ＝Olive Young 入口」に統一するかで選択肢あり。 |
| **自然な日本語** | キーワード詰め込みより、説明文として読みやすい短文を優先（現行どおり）。 |

### B. ユーザー導線

| 遷移先 | 役割 |
|--------|------|
| **商品ページ** | メイン導線。ランキングTOP3・ランキング一覧・カテゴリ一覧・ブランド詳細から「商品詳細」へ。商品詳細で summary / review / ingredient / price を表示し、購入導線へ。 |
| **ブランドページ** | ブランド単位のランキング・人気商品。急上昇/NEWブランドから遷移。 |
| **カテゴリページ** | マスク・美容液・クリーム別ランキング。ニッチな検索・比較ニーズの受け皿。 |
| **ランキング一覧** | 全順位の商品一覧。今日の注目 TOP3 の次に「もっと見る」として有効。 |

### C. 収益導線

| ポイント | 内容 |
|----------|------|
| **主導線** | トップ → 商品詳細 → 「購入・価格を確認する」（検索リンク＋Olive Young で見る）・価格比較セクション。クリックは商品詳細以降で発生させる設計で問題なし。 |
| **トップでの役割** | ランキング・ブランド・カテゴリへの「入口」を明確にし、離脱せず商品詳細まで届ける。トップ自体に広告やアフィリエイトを並べるより、まずは導線の整理を優先。 |

## 3. トップページの構成案（既存データの範囲）

「今あるデータだけで実現できる」範囲で、現行 `/oliveyoung` をベースにした案。

| ブロック | 内容 | データ元 | 備考 |
|----------|------|----------|------|
| **ヒーロー** | タイトル・短文説明・対象日・更新頻度 | 既存 | 現行維持。キーワードは自然に。 |
| **カテゴリリンク** | マスク / 美容液 / クリーム | CATEGORY_SLUGS | 現行維持。 |
| **CTA** | 商品ランキングを見る / ブランドランキングを見る | runDate | 現行維持。 |
| **サマリー** | 対象日・総商品数・総ブランド数・NEW数 | getRankingByDate, getBrandRankingByDate | 現行維持。 |
| **注目商品** | TOP3 または TOP5～10 | getRankingTopNWithProducts | 件数だけ増やす選択肢あり。画像・名前・商品詳細リンク。 |
| **急上昇 / NEWブランド** | 各 TOP5 | getBrandRankingByDate | 現行維持。 |
| **導線カード** | 商品ランキング一覧 / ブランドランキング一覧 | runDate | 現行維持。 |
| **カテゴリ別ショートカット** | マスク・美容液・クリームの「人気TOP3」など | 既存ランキング＋カテゴリ判定 | 将来: カテゴリ別TOPを取得するAPI/関数があれば追加。無ければ「カテゴリ一覧リンク」のみで十分。 |

**新規データがなくてもできること**

- 注目商品を TOP3 → TOP5 や TOP10 に増やす（getRankingTopNWithProducts の N を変えるだけ）。
- サイトルート `/` を `/oliveyoung` にリダイレクトし、「トップ＝メディア入口」をはっきりさせる。
- 見出しや説明文を「収益化メディアの入口」に合わせて微修正（文言のみ）。

**新規データが欲しくなるもの（将来）**

- カテゴリ別「TOP3」をトップに載せる（現状はカテゴリページへのリンクのみ）。
- 「新着」「再入榜」など別軸のセクション（対応する crawler/集計があれば追加）。

## 4. 実装優先順位

| 優先度 | 内容 | 理由 |
|--------|------|------|
| **1** | **サイトルート `/` の扱いを決める** | 現状プレースホルダーのままなので、リダイレクト `/` → `/oliveyoung` にするか、最小のランディング（タイトル＋Olive Young 入口へのリンク1本）にするかで設計を確定させる。 |
| **2** | **`/oliveyoung` を「メディア入口」として明示する** | title/description やヒーロー文言を「韓国コスメ人気ランキングの入口」のように少し整える。既存ブロックの並びはそのままでよい。 |
| **3** | **注目商品の件数** | TOP3 のままでよいが、余裕があれば TOP5 に増やして商品詳細への導線を少し強化。 |
| **4** | **過剰にしないライン** | トップに特集・比較記事・多数のブロックを増やさない。まずは「ランキング・ブランド・カテゴリへの入口」と「注目商品→商品詳細」の流れを明確にすれば十分。 |
